# LLM Wiki Project Base

Filesystem starter for a new [LLM Wiki](https://github.com/nashsu/llm_wiki) knowledge base project.

This folder documents the **shared layout** used by every wiki instance (including [work-wiki](https://github.com/zohuyhieuzo03/work-wiki)). The actual scaffold is generated by `scripts/new-wiki.sh` or the desktop app's **Create Project** dialog — both use the same `src/lib/project-scaffold.ts` + `src/lib/templates.ts` source of truth.

## Shared layout (llm_wiki app ↔ work-wiki)

| Layer | Path | Role |
|-------|------|------|
| Schema | `schema.md` | Page types, naming, frontmatter, ingest/query/lint rules |
| Purpose | `purpose.md` | Goals, scope, open questions — the wiki's intent |
| Raw | `raw/sources/` | Immutable inputs (never edited by LLM) |
| Raw assets | `raw/assets/` | Images, attachments |
| Catalog | `wiki/index.md` | LLM navigation entry — read first |
| Changelog | `wiki/log.md` | Append-only log (`## [date] \| action \| title`) |
| Overview | `wiki/overview.md` | High-level state summary |
| Typed pages | `wiki/{entities,concepts,...}/` | Synthesized knowledge |
| Agent skills | `.agents/skills/wiki-{ingest,query,lint}/` | Cursor agent workflows (no desktop app required) |
| App state | `.llm-wiki/` | Desktop app metadata (gitignored) |

## Three operations (Karpathy pattern)

1. **Ingest** — source → raw snapshot → wiki pages → index + log + cross-links
2. **Query** — search wiki; file durable answers back (not chat-only)
3. **Lint** — orphans, broken wikilinks, index drift, stale claims

## Quick start

```bash
# From llm_wiki repo root
chmod +x scripts/new-wiki.sh
./scripts/new-wiki.sh my-wiki ~/wikis/learning business

cd ~/wikis/learning/my-wiki
# Edit purpose.md, then open in LLM Wiki app or Cursor
```

Or use **LLM Wiki desktop → Create Project** and pick a template — same scaffold files are written automatically.

## Templates

| ID | Use case |
|----|----------|
| `general` | Blank slate |
| `business` | Team / engineering (meetings, ADRs, projects, slides) — **work-wiki uses this** |
| `research` | Hypothesis, methodology, findings |
| `reading` | Books — characters, themes, chapters |
| `personal` | Goals, habits, journal |

## work-wiki extensions

[work-wiki](https://github.com/zohuyhieuzo03/work-wiki) is a **Business** instance with domain content (VMS engineering). It adds:

- `wiki/slides/` — Marp decks (now in Business template)
- Parseable log actions (`ingest`, `lint`, `query`, …)
- Richer `schema.md` conventions (ticket IDs, investigation structure)
- Optional `scripts/start-llm-wiki.sh` for local desktop app

New wikis get the same base; customize `purpose.md` and ingest domain sources.
